Build a complete, polished, responsive web application called **Daleel | دليل**.

Daleel is an AI-powered platform that helps users understand contracts, subscriptions, terms and conditions, and service agreements before accepting or signing them.

The user uploads a PDF, image, or pasted contract text. The AI analyzes it and converts it into a clear, simple report showing important terms, costs, dates, cancellation rules, automatic renewal, penalties, refunds, privacy clauses, obligations, and potential risks.

The application must be functional, not just a static design or landing page.

---

# Core Product Experience

The main user journey should be:

1. User opens the website.
2. User uploads a PDF or image of a contract, or pastes contract text.
3. User selects the contract type.
4. The AI analyzes the contract.
5. The user sees a clear structured report.
6. The user can ask questions about the contract.
7. The user can save, download, compare, or create reminders from the contract.

The application should support:

* PDF contracts.
* Scanned contract images.
* JPG, PNG, and WEBP images.
* Pasted text.
* Arabic contracts.
* English contracts.
* Contracts containing both Arabic and English.

# Risk Levels

Classify important clauses using three levels:

## Low

Normal information the user should know.

Examples:

* Payment is due on the first day of every month.
* Cancellation requires written notice.

## Medium

A clause that may create an additional commitment.

Examples:

* The contract automatically renews.
* The company may modify the service price.

## High

A clause that may cause financial loss or significantly limit the user’s rights.

Examples:

* A large early cancellation penalty.
* No refund under any circumstances.
* Automatic renewal for another full year.
* Broad permission to share personal data.

Use colors carefully:

* Green for low.
* Amber for medium.
* Red for high.

Do not describe a clause as illegal. Use wording such as:

* “This clause may require additional attention.”
* “Consider reviewing this clause carefully.”
* “You may want to ask the provider for clarification.”

# Source Verification

Every important extracted clause must include:

* Page number when available.
* Original text excerpt.
* A button called “View source”.
* A highlighted source area when possible.

The application must not invent page references.

When a source page cannot be determined, display:

“Source location could not be identified.”

The user should always be able to compare the AI explanation with the original contract.

# Ask Daleel

Create a chat-style page called:

**اسأل دليل**

The user can ask questions about the uploaded contract.

Examples:

* هل أقدر ألغي العقد بعد شهرين؟
* كم غرامة الإلغاء؟

* هل يحق للشركة رفع السعر؟

Responses must:

* Be based only on the uploaded contract.
* Be short and clear.
* Mention the relevant clause.
* Include the source page when available.

Example:

“يسمح العقد بالإلغاء المبكر، لكن توجد غرامة قدرها 500 ريال. راجع البند 8 في الصفحة الرابعة.”

“لم أجد إجابة واضحة على هذا السؤال داخل العقد المرفوع.”

# Contract Clarity Score

Create a clarity score from 0 to 100.

The score may consider:

* Sentence complexity.
* Length of clauses.
* Number of unclear terms.
* Scattered financial information.
* Missing cancellation details.
* Excessive legal terminology.
* Conflicting clauses.

Display:

* Overall score.
* Explanation of the score.
* Areas that need clarification.

Example:

**Clarity score: 62/100**

“Cancellation terms are spread across multiple sections, and some additional fees are not clearly explained.”

Make it clear that this score evaluates readability and clarity, not legal validity.
